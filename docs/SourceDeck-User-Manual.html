§ 2 · Integrations & Criteria
03 · Adding API keys & integrations

Your keys. Your workspace. Nothing shared.

SourceDeck never ships with keys pre-installed. Every operator brings their own credentials; every workspace is isolated.

Required · Airtable PAT (base-scoped) · Instantly API key · Email sender domain
Optional · Apollo key · SerpAPI key · Hunter key · OpenAI / Anthropic key
  • Keys are stored locally per workspace. They never leave the device unless an integration call is in flight.
  • Workspace reset (see p.7) wipes every key, cached lead, and saved search in one action.
  • Each key is namespaced so switching workspaces swaps keys atomically — no cross-contamination.
04 · Setting lead criteria

Tell SourceDeck who counts.

Define the filters the discovery engine enforces on every run. Narrow criteria + tight email validation = clean queue.

Geography
Pick cities, metro areas, or states. Multi-region rotation supported.
Industry / NAICS
Map target industries to NAICS. Multi-industry allowed per campaign.
Exclusions
Block role inboxes, free-mailbox domains, and prior-contacted fingerprints.
Lead quality rule
Discovery should never produce leads that couldn't survive a MX check. If bounces exceed 3% of the last 50 sends, pause and tighten criteria before resuming.

§ 3 · Campaigns
05 · Building and launching a campaign

From empty canvas to live sequence.

Every campaign has three parts: a sender, a sequence, and a routing filter. Build each one deliberately.

Sender
Attach at least one warm mailbox
A warmed sender protects deliverability. Add a second when daily cap exceeds 25.
Sequence
4-step diagnosis → pricing
Email 1 diagnoses. Email 2 exposes the pattern. Email 3 names the system. Email 4 closes on pricing.
Filter
MX-verified, non-role, non-free
Only leads with MX passed, no role prefix, and no consumer domain are enrolled.
Cap
Start at 25/day per sender
Ramp by +10 per day if bounces stay under 2% over the last 50 sends.

Safe-resume posture

  • Campaign starts paused. Enroll leads, verify sequence previews, then activate.
  • Stop-on-reply is on by default. Replies never get a second touch.
  • Link tracking is on only for the pricing-CTA email so intent is measurable without noise.
  • Open tracking is off. Clean, text-only sends look like humans, not tools.
Operator discipline
If bounce rate crosses 3% on the last 50 sends, pause immediately. Audit the queue. Resume only after the rate drops below 2% on the next fresh batch.

§ 5 · Reset & Support
08 · Resetting a workspace

One click, clean slate.

Reset wipes every workspace artifact — keys, leads, cached analytics, saved searches, templates, and event history. The next launch opens in the same blank state a first-time user sees.

  • Workspace reset clears localStorage, sessionStorage, and cached analytics associated with this workspace.
  • Campaign IDs and sender credentials are forgotten; the next workspace must supply fresh keys.
  • Demo mode exists for evaluation only and is never auto-loaded into a live workspace. It cannot pollute production data.
  • Every install is sanitized — no prior operator data is ever shipped with SourceDeck.
Customer data protection
SourceDeck treats every workspace as a sealed unit. Keys and leads never leave the device without an explicit API call. Exports are opt-in. Logs are workspace-scoped.
