Scope and setup rules

Every plan is delivered inside a defined scope. These rules apply to all tiers unless separately quoted.

1 · Approved tool connections

Each plan only covers the number of approved tool connections listed in that plan. An approved tool connection means a standard supported integration such as OpenAI, Claude, Zapier, Make, Airtable, Notion, Gmail, Google Drive, Slack, or a basic webhook-supported system.

2 · What counts as one connection

One connection means one supported platform connected for the agreed workflow.

  • OpenAI connected to SourceDeck = 1 connection
  • Zapier connected to SourceDeck = 1 connection
  • Airtable connected to workflow = 1 connection

3 · Not included in standard setup

These items are outside standard plan scope unless separately quoted:

  • Custom API development
  • Unsupported or niche software
  • CRM rescue work
  • Broken legacy automation cleanup
  • Advanced branching logic beyond standard delivery
  • Multi-team deployment
  • Multi-brand deployment
  • Ongoing campaign execution
  • Copywriting retainers
  • Lead generation services
  • Sales operations management

4 · API keys and private credentials

Clients maintain control of their own API keys, logins, billing relationships, and private credentials. Where required, we guide you through entering those secrets into your environment. We do not assume responsibility for third-party platform billing, usage fees, suspensions, or account restrictions.

5 · Revision limits

Standard delivery includes one implementation pass within the purchased scope. Anything beyond the agreed setup, approved integrations, or original workflow is handled as additional scope.

6 · Support windows

Support windows apply only to issues directly related to the delivered SourceDeck implementation. They do not include:

  • Third-party outages
  • Platform policy changes
  • Client-side account problems
  • New feature requests
  • Unrelated consulting
